I. Создание .venv для проекта

___

**I. Создание .venv для проекта**
--

# Создание виртуального окружения

python -m venv myenv

# Активация

source myenv/bin/activate   # MacOS, Linux
myenv\Scripts\activate      # Windows

pip show название_библиотеки    # Показывает установлена указанная библиотека в .venv
pip install название_библиотеки # Устанавливаем указанную библиотеку
pip install ipykernel          # Устанавливаем пакет для ядра Юпитера Питона

deactivate                     # Завершаем работу с виртуальным окружением (если он больше не нужен)

# Установка зависимостей

pip install -r requirements.txt

**II. Cпособы минимизировать различия и облегчить перенос проекта между системами:**
--

**1. Используйте Docker контейнеризацию**
--

Docker позволяет упаковать ваше приложение вместе с его зависимостями и средой исполнения в единый образ контейнера. Такой контейнер будет запускаться одинаково независимо от базовой операционной системы (Windows, macOS, Linux):
Создавайте Dockerfile, определяйте необходимые зависимости и шаги установки.
Запускайте приложение внутри контейнера, избавившись от проблем совместимости ОС.
Пример Dockerfile для Python-приложения:

FROM python:3.10-slim-buster

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY . .

CMD ["python", "./your_app.py"]

Затем создайте образ и запустите контейнер:

docker build -t your_project .
docker run -it --rm your_project

**2. Общие требования (requirements.txt)**
--

Используйте общий файл требований (requirements.txt), содержащий список необходимых модулей и их версий. Этот файл позволит легко устанавливать нужные библиотеки в любое виртуальное окружение на любой платформе:

pip freeze > requirements.txt

При переходе на новую ОС достаточно воссоздать виртуальное окружение и установить требуемые зависимости:

# Создание виртуального окружения

python -m venv env

# Активация

source env/bin/activate   # MacOS, Linux
env\Scripts\activate      # Windows

# Установка зависимостей

pip install -r requirements.txt

**3. CI/CD (Continuous Integration & Continuous Deployment)**
--

Настройте автоматическое тестирование и сборку проекта на всех целевых платформах (например, используя GitHub Actions, Jenkins или Travis CI). Автоматизация сборки позволит убедиться, что проект стабильно работает на разных ОС перед деплоем.
Эти подходы позволяют эффективно управлять проектом и избегать привязанности к определенной операционной системе.
